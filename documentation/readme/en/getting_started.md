## This tutorial refers to the English language

# Getting Started

In the file [README](/README.md) you will find an explanation of what Config Editor is, its features, installation methods, and a basic configuration example to run the container.

In this tutorial, we will go deeper into the configuration options available during installation.

### Container name

Defines a name for your container (can be anything).

Example:
``` yaml
container_name: config-editor
```

### user: "0:0"

This parameter is required to allow interaction with the docker.sock.

With it, Config Editor can inspect containers and execute the `Restart container` action directly from the interface.

### Ports

By default, the application runs on port 5000 inside the container.

You can expose it on any external port you prefer.

Example:
``` yaml
ports:
  - "8080:5000"   # accessible at http://host:8080
```

### Flask secret key

Insert here a random and secure secret key used for session encryption.

This key should be generated by the administrator before starting the container.

Example of generation:
```bash
python - <<'PY'
import secrets, base64
print(base64.urlsafe_b64encode(secrets.token_bytes(48)).decode())
PY
```

### TOTP_ENABLED

Defines whether two-factor authentication (2FA) will be active in the system.

- `true` → you will be required to set up 2FA at first login (via Google Authenticator, Authy, etc).
- `false` → initial registration happens with only username and password.

### FILE_CONTAINERS

Specifies which configuration files the editor should manage and to which container they belong.

Format:
```yaml
FILE_CONTAINERS: "arquivo.xml:nome_do_container"
```

Example with multiple files:
```yaml
FILE_CONTAINERS: "traccar.xml:traccar,tunnel.yml:cloudflared"
```

### DOCKER_HOST

Defines the path to the Docker socket that will be used to inspect and restart containers.

Default value:
```perl
unix:///var/run/docker.sock
```

### DATA_DIR

Directory inside the container used to read and write configuration files.

Example volume:
```yaml
- /srv/my_services/config:/data
```

### BACKUP_DIR

Directory used to store backups created via Config Editor.

Example:
```yaml
- /srv/config-editor/backups:/backups
```

### STATE_DIR

Directory where Config Editor stores its internal data (registered users, 2FA secrets, etc).

Example:
```yaml
- /srv/config-editor/state:/state
```

### volumes

- /var/run/docker.sock:/var/run/docker.sock

Mandatory volume so Config Editor can communicate with Docker.

Without it, inspecting containers and using the Restart container button will not work.

- /srv/service/config:/data

Defines where your service configuration files will be accessible inside Config Editor.

⚠️ Important: always bind the folder containing the files, not a single file.
  - ✅ Correct: /srv/service/config:/data
  - ❌ Wrong: /srv/service/config/arquivo.xml:/data/arquivo.xml

If you have multiple services in different folders, map each folder separately.

- /srv/config-editor/backups:/backups

Defines where backups created via the interface are stored.

Even if you remove or restart the container, backups will remain in this host folder.

- /srv/config-editor/state:/state

Stores Config Editor’s internal state (users, 2FA keys, etc).

This ensures that even if you recreate the container, the configuration is not lost.

## Optional environment variables

### DEBUG (default: false)

Enables or disables Flask debug mode.

- `false` → recommended for production.
- `true` → useful only for development/testing (shows detailed errors in the browser).

### DRY_RUN (default: false)

Controls Config Editor’s Dry-Run mode.

- `false` → Save & Apply actions really save and apply changes.
- `true` → changes are validated but not applied (useful for testing without impacting containers).

### DIFF_ALLOW_EDIT (default: false)

Controls whether the diff view allows editing.

- `false` → you can only view differences.
- `true` → enables editing directly in the diff view.
